# 可迁移学术写作规律

## 用途与边界

本文件只指导如何组织和表达当前任务的证据。它不是文献、理论库或经验证据，不能支持任何事实性论断。当写作规律与当前文献、目标期刊或研究设计冲突时，以当前证据和体裁要求为准。

## 绪论：用论证关系完成漏斗收束

核心链条是：

`问题为什么重要 → 已经知道什么 → 哪项解释或推断仍然失败 → 本研究如何修复 → 理论与实践贡献`

- 把“漏斗”理解为论证范围逐步收束，不要套用固定段数、固定句式或宏大背景开头。
- 第一段可从现实变化、反常现象或重要问题切入，但必须把现实问题翻译为学术对象。市场规模或社会热度不能代替知识问题。
- 在提出缺口前建立“知识基线”：已有研究解释了什么、通过什么证据解释、适用于什么边界。
- 缺口要指向理论、机制、边界、构念、测量或可信推断的具体失效位置，不要从“很少”直接跳到“必须研究”。
- 让研究问题或目标逐项回应缺口。复杂竞争解释宜显式提出研究问题；直接且界定清晰的任务可用连续目的陈述收束。
- 把贡献写成对缺口的回答，并确保讨论部分能逐项兑现。无法在设计和结果中兑现的承诺应删除或降级。
- 篇章预告只是长文或特定期刊的可选模块。不要为了形式完整而机械加入。

## 系统综述：从文献分类转向解释综合

解释型综述优先使用：

`理论定位 → 构念比较 → 机制整合 → 分歧与边界 → 综合判断`

每节优先完成：

`本节问题 → 主要共识 → 同向证据 → 分歧/反例 → 可能解释 → 情境边界 → 与下一节的关系`

- 不按文献逐篇摘要；先形成本节要回答的问题，再按证据对该问题的功能组织文献。
- NMF主题可帮助定位材料，但不必然等于最佳叙事结构。必须通过内容证据重新命名，并解释主题之间的互补、竞争、嵌套、迁移或断裂。
- 时间演进适用于概念史、理论史、技术史和测量工具史；不要把年代顺序当成所有综述的默认骨架。
- 主题分类适合建立领域地图，但各类之间必须有可说明的比较维度和关系。
- 方法和测量分类用于解释结论为何不同、哪些推断可信。不要把方法清单与理论主题混成同一层级。
- 每个段落只承担一个主要判断，并以综合、限定或过渡收束，不以文献清单结束。

## 段落与原子论断

默认段落单元是：

`主题判断 → 精确证据组 → 异质性/反例 → 解释与限定 → 综合过渡`

- 一个句子同时包含多个对象、机制、媒介或结果时，拆成最小可验证论断，将文献放在它真正支持的论断之后。
- 定义、理论来源和首创判断优先使用明确的作者型引文；共识和同质结果可使用聚合引文。
- 正向、反向和不显著证据要分别绑定，不用同一个模糊的段末引文串同时支持相反结论。
- 同一文献可支持多个论断，但每次都要重新限定支持对象、方向、范围和证据等级。
- 引用数量不能补偿语义不匹配。每个实质性知识主题段落都应有直接相关文献，不只解释计量主题词。

## 理论的角色与分工

将理论区分为：

- **总框架**：负责整体解释顺序和分析层级。
- **局部机制理论**：解释某一条路径或心理过程。
- **边界理论**：说明解释何时、对谁或在什么情境下失效。
- **构念来源**：只负责定义、概念谱系和测量边界。
- **竞争理论**：提供可以通过设计区分的替代解释。

理论名称出现不等于研究真正使用或检验了该理论。多理论写作先说明分工、层级和互补或竞争关系，再展开路径。理论整合必须增加解释力或产生可区分预测；否则删除不必要的理论。没有适配理论时，改用“机制—边界—竞争解释”的理论中性叙事。

## Gap—设计—贡献闭环

固定检查链是：

`已有知识 → 当前解释 → 失效位置 → 知识后果 → 新解释需求 → 可检验问题 → 识别性设计 → 贡献`

- “研究较少”“首次研究”“更换国家/平台”“未使用某方法”只是候选信号，不能直接支持核心贡献。
- 结果矛盾先检查构念、结果变量、测量、样本、时间、情境和设计是否可比。
- 机制缺口先确认基础关系有可用证据，再排查同义机制、竞争机制和“只报告统计中介”的情形。
- 情境扩展必须指出新情境改变了哪一项理论假设、行为约束或外推条件。
- 先选择能识别目标问题的研究设计，再选择估计或分析工具。方法复杂度不构成贡献。
- 把每个缺口映射到研究问题、假设/命题、设计、数据、方法、可支持的结论和贡献。任一环无法对应时，缩小措辞或重新设计。

## 体裁适配

- **实证论文绪论**：问题化 → 知识基线 → 解释/推断缺口 → 研究任务 → 贡献。
- **系统综述**：范围与概念边界 → 知识结构 → 分歧与异质性 → 解释 → 研究议程。
- **方法论文**：常见误用 → 现有诊断不足 → 新方法/工作流 → 适用条件与限制。
- **测量研究**：构念边界 → 已有工具覆盖 → 测量失配 → 开发与验证任务。
- **算法/计算研究**：任务瓶颈 → 现有方法族 → 性能/可解释性/泛化限制 → 解决方案与边界。
- **定性/过程研究**：现象问题化 → 已有解释不足 → 过程未知 → 材料/案例选择 → 概念或理论生成目标。

体裁不明时，先根据研究问题、证据形式和交付类型判定，不默认套用实证量化论文结构。

## 使用顺序

1. 在提纲前确定体裁和论证任务。
2. 为每节写一份论证契约，再建立原子论断—证据表。
3. 写作时使用当前任务的 dossier、claim ledger和证据卡，不从本协议推导事实。
4. 分节检查综合、理论分工、引文归属和过渡，再合并全文。
5. 最终检查缺口、目标、设计和贡献是否闭合，同时保持引用、因果和参考文献的硬性审计标准。
